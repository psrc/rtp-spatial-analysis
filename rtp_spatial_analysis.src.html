rtp_spatial_analysis.src.density_and_freight module¶

Density and Freight Analysis Module¶

This module provides functionality for analyzing activity unit density within proximity of Freight and Goods Transportation System (FGTS) routes in the Regional Transportation Plan.

The analysis involves:

  • Buffering FGTS polyline routes (T-1 and T-2 classifications)

  • Intersecting the buffered zones with activity unit hex grid layers

  • Calculating and summarizing activity units within the buffer zones

  • Comparing regional totals against values within freight corridors

Example

To run the full density and freight analysis:

from rtp_spatial_analysis.src import density_and_freight
from rtp_spatial_analysis.src.configuration import Config

config = Config().config
density_and_freight.run(config)

Buffer a polyline layer and intersect it with a hex grid layer.

Creates a 500-foot buffer around the input polyline features and performs a spatial intersection with the provided hexagonal grid layer. This is commonly used to identify activity units that fall within proximity of transportation routes.

Parameters:

  • hex_layer (geopandas.GeoDataFrame) – A GeoDataFrame containing hexagonal grid polygons with activity unit data to intersect with the buffer.

Returns:

A GeoDataFrame containing the intersection of the buffered polylines and the hex grid, preserving attributes from both layers.

Exception – If buffering or intersection operations fail due to invalid geometries or other spatial processing errors.

rtp_spatial_analysis.src.density_and_freight.run(config)¶

Execute the density and freight analysis workflow.

This is the main entry point for the density and freight analysis. It calculates activity unit density within 500 feet of FGTS (Freight and Goods Transportation System) routes classified as T-1 or T-2.

The workflow performs the following steps:

  1. Loads FGTS route data and filters for T-1 and T-2 classifications

  2. Loads activity unit data for both 2050 (projected) and 2024 (current) years

  3. Creates 500-foot buffers around FGTS routes

  4. Intersects buffers with activity unit hex grids

  5. Calculates regional totals and buffer-specific totals

  6. Exports results to a CSV summary table

Parameters:

config (dict) –

Configuration dictionary containing file paths and settings. Required keys include:

  • 'fgtswa_path': Path to FGTS routes layer

  • 'activity_units_path': Path to activity units data

  • 'acre_in_sqft': Conversion factor for area calculations

  • Output path settings for CSV export

Returns:

None. Results are exported to density_and_freight.csv.

Return type:

None

Raises:

  • FileNotFoundError – If input data files specified in config cannot be found.

  • KeyError – If required configuration keys are missing.

  • Exception – If spatial processing or data export operations fail.

Output Format

The exported CSV contains the following columns:

Column

Description

selection

Either ‘regional total’ or ‘within 500 ft of FGTS routes’

activity units 2050

Projected activity units for 2050

activity units 2024

Current activity units for 2024

See also

buffer_and_combine(), sum_combined(), utils.export_csv()

rtp_spatial_analysis.src.density_and_freight.sum_combined(gdf, au_col_name, config)¶

Calculate the sum of activity units in a GeoDataFrame.

Computes the total activity units by:

  1. Extracting relevant columns including geometry and activity units per acre

  2. Calculating the acreage of each polygon from its geometry

  3. Multiplying activity units per acre by acreage to get total activity units

  4. Summing all activity units across the GeoDataFrame

Parameters:

  • gdf (geopandas.GeoDataFrame) – A GeoDataFrame containing polygon geometries with activity unit density data. Must include columns for GRID_ID, the activity unit column specified by au_col_name, au_acre (activity units per acre), and geometry.

  • au_col_name (str) – The name of the column containing activity unit values (e.g., 'sum_au_205' for 2050 data or 'sum_au_202' for 2024 data).

  • config (dict) – Configuration dictionary containing conversion factors. Must include the key 'acre_in_sqft' specifying square feet per acre.

Returns:

The total sum of activity units across all polygons in the GeoDataFrame.

Return type:

float

Raises:

  • KeyError – If required columns are missing from the GeoDataFrame or if 'acre_in_sqft' is not in the config dictionary.

  • Exception – If calculation fails due to invalid data types or other processing errors.

Note

NaN values in au_acre and acreage columns are filled with 0 before multiplication to prevent null results.

rtp_spatial_analysis.src.utils.buffer_layer(layer_gdf, distance)¶

Create a buffered copy of a GeoDataFrame’s geometries.

Applies a buffer operation to all geometries in the input GeoDataFrame, expanding or contracting them by the specified distance.

Parameters:

  • layer_gdf (geopandas.GeoDataFrame) – The input GeoDataFrame containing geometries to buffer.

  • distance (float) – The buffer distance in the units of the GeoDataFrame’s coordinate reference system. Positive values expand geometries, negative values contract them.

Returns:

A copy of the input GeoDataFrame with buffered

geometries. Returns unchanged copy if distance is 0 or negative.

Return type:

geopandas.GeoDataFrame

Raises:

Exception – If an error occurs during the buffer operation.

rtp_spatial_analysis.src.utils.get_onedrive_layer(config, path_name, layer)¶

Load a specific layer from a geodatabase file stored in OneDrive.

This function constructs a file path using configuration settings to locate and load a geodatabase layer from OneDrive storage. The loaded data is automatically reprojected to EPSG:2286 (Washington State Plane North).

Parameters:

  • config (dict) – Configuration dictionary containing OneDrive path information. Must include ‘user_onedrive’ key and the key specified by path_name.

  • path_name (str) – Key name in the config dictionary that contains the relative path to the geodatabase file from the OneDrive root.

  • layer (str) – Name of the specific layer to read from the geodatabase file.

Returns:

EPSG:2286 (Washington State Plane North).

geopandas.GeoDataFrame

rtp_spatial_analysis.src.utils.points_in_polygon(points_gdf, polygons_gdf, col_name, buffer=0)¶

Check if points intersect polygons and add a boolean column to the result.

Determines whether each point in the points GeoDataFrame intersects with any polygon in the polygons GeoDataFrame. Optionally applies a buffer to the points before checking intersection.

Parameters:

  • points_gdf (geopandas.GeoDataFrame) – A GeoDataFrame containing point geometries to check for intersection.

  • polygons_gdf (geopandas.GeoDataFrame) – A GeoDataFrame containing polygon geometries to check against.

  • col_name (str) – The name of the boolean column to add to the points GeoDataFrame indicating intersection status.

  • buffer (float, optional) – Buffer distance to apply to points before checking intersection, in the units of the coordinate reference system. Defaults to 0 (no buffer).

Returns:

The input points GeoDataFrame with an additional

boolean column indicating whether each point intersects the polygons.

Return type:

geopandas.GeoDataFrame

Raises:

Exception – If an error occurs during the intersection check.
